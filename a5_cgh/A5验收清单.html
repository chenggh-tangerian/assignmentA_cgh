2. 实验对比（同模型 · 同数据集）

统一条件：模型 rl/outputs/train_lora_ppo_full/checkpoint-1304， 评测集 MBPP sanitized test（257 题），对比三种本人负责策略的准确率 vs Token 成本 vs 墙钟耗时。

策略 pass@1 通过题数 syntax avg tokens / task avg 耗时 / task LLM 调用 / task
Greedy 12.45% 32 / 257 82.9% 278 2.1 s 1
CoT 10.12% 26 / 257 81.3% 379 3.4 s 1
ToT 6.23% 16 / 257 98.4% 4,681 26.1 s ~9
对比结论
  • Greedy 准确率最高（12.45%），成本最低（278 tokens / 2.1 s），作为 TTS 成本下界
  • CoT 未带来 pass@1 提升（10.12% < Greedy），但输出更长（+36% tokens），可能因 Final code 抽取失败或推理链干扰
  • ToT 语法率最高（98.4%），计划→代码两阶段有效；但 pass@1 最低且成本约为 Greedy 的 17× tokens / 12× 耗时
  • 在当前 PPO checkpoint 上，简单 greedy 解码优于复杂 Test-Time 搜索，性价比需按场景权衡

3. 独立输入输出

Greedy

Input

  • MBPP 任务行（prompt + test_list）
  • 官方 MBPP Prompt 模板（[BEGIN]/[DONE]）
  • 模型 checkpoint 路径

Output

  • mbpp_metrics_greedy.json — pass@1、token、timing
  • mbpp_cases_greedy.jsonl — 逐题代码 + 测试结果
  • mbpp_greedy_records.jsonl — 原始 completion
CoT

Input

  • MBPP 任务描述 + 测试用例
  • CoT 结构化 Prompt（Reasoning / Key steps / Final code）
  • 模型 checkpoint 路径

Output

  • mbpp_metrics_cot.json
  • mbpp_cases_cot.jsonl — 含完整 CoT 回复 + 抽取代码
  • mbpp_cot_records.jsonl
ToT — 含调用轨迹

Input：MBPP 任务 + 搜索超参（branch_factor / beam_width / depth）

Output：

  • mbpp_metrics_tot.json — 汇总指标
  • mbpp_tot_records.jsonl — 每题完整搜索记录
  • mbpp_nodes_tot.jsonl — 调用轨迹：thought 节点 → code 候选 → 打分 → 选优路径
task prompt → 3× thought 提案 → beam 剪枝 (2) → 2×2 code 展开 → verifier 打分 → 最优 code

7. 模块说明

7.1 Greedy 基线

MBPP 官方 Prompt（[BEGIN] / [DONE]），temperature=0 单次解码，每题 1 次 LLM 调用。

  • 代码抽取：extract_candidate_code()（mbpp_eval_dpo）
  • 全量结果：pass@1 12.45%，278 tokens/task，2.1 s/task

7.2 CoT 基线

Reasoning:   — 解题思路
Key steps:   — 实现步骤
Final code:  — ```python ... ```  （仅此区块参与评测）
  • 默认：无 few-shot，greedy 解码，max_new_tokens=768
  • 全量结果：pass@1 10.12%，379 tokens/task，3.4 s/task

7.3 Tree of Thoughts

两阶段 beam 搜索（BF=3, BW=2, D=2）：计划提案 → 剪枝 → 代码展开 → verifier 选优。

  • 全量结果：pass@1 6.23%，4681 tokens/task，26.1 s/task，syntax 98.4%
  • ToT Token 分解：thought 307K + code 895K = 合计 1.2M tokens
