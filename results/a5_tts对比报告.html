1. 实验设置与可比性说明

统一条件
  • 模型：rl/outputs/train_lora_ppo_full/checkpoint-1304
  • 评测集：MBPP sanitized test，257 题，zero-shot prompt（example task IDs: 2,3,4）
  • 评测底座：共用 dpo/scripts/mbpp_eval_dpo.py（prompt 构造、代码抽取、assert 执行）
  • Greedy 基线：MBPP 官方模板 + temperature=0 单次解码，作为所有 TTS 策略的成本下界
可比性要点：Greedy / CoT / ToT（a5_cgh/）与 SC / BoN / Reflexion（a5/）在 prompt 构造、chat template、执行评测上对齐同一底座。 Greedy 每题 input token = 148.9，与 a5 多采样策略的单次生成 token 完全一致，验证了 prompt 一致性。 CoT 使用结构化 Prompt（Reasoning / Key steps / Final code），与 Greedy 的 MBPP 官方模板不同，属于 Prompt 结构对比而非纯 TTS 对比。

三句话总结
  1. Best-of-N 准确率最高（23.35%），相对 Greedy 提升 +10.9 pp，syntax 达 99.6%，但全量生成成本 ≈ Greedy 的 8.2×。
  2. Greedy 基线本身已不弱（12.45%），优于 CoT greedy（10.12%）、ToT（6.23%）、SC（7.39%），说明在此 checkpoint 上 MBPP 官方 prompt + T=0 解码是强基线。
  3. Reflexion 性价比较均衡：pass@1 8.56%（+2.3 pp over initial），Token ≈ Greedy 的 10×，耗时 ≈ Greedy 的 7.3×，按需触发仅失败题额外调用。

4. Oracle 上界与选优 gap

多采样/搜索策略的 pass@k（oracle）反映「生成池里有没有正确答案」，pass@1 与 pass@k 的 gap 反映选优策略的损失。

策略 pass@1（选优后） pass@k / pass@n（oracle） Oracle gap 解读
Self-Consistency 7.39% 24.90% (64/257) 17.5 pp 10 条路径中有 64 题存在正确解，但投票仅选出 19 题 → 选优损失大
Best-of-N 23.35% 23.35% (60/257) 0 pp 8 候选中最好的即为最终答案，无 oracle gap
ToT 6.23% 6.23% (16/257) 0 pp beam 选优无损失；瓶颈在搜索/generation 而非 rerank
Reflexion 8.56% — — initial 6.23% → final 8.56%，6 题被反思挽救（+2.3 pp）

6. 性价比分析

策略 pass@1 提升 vs Greedy Token 倍数 vs Greedy 耗时倍数 vs Greedy pass@1 增益 / Token 倍数 推荐场景
Best-of-N +10.9 pp 8.2× 11.0× 1.33 pp/× 准确率优先，可接受 8× 生成成本
Reflexion −3.9 pp* 10.0× 7.3× — 失败题按需反思；*initial 非 greedy 口径
Self-Consistency −5.1 pp 10.2× 13.2× 负收益 oracle 24.9% 说明采样有效但投票策略需优化
ToT −6.2 pp 16.8× 12.3× 负收益 syntax 98.4% 高，适合语法约束场景；pass@1 需改进 thought 质量
CoT −2.3 pp 1.4× 1.6× 负收益 此 checkpoint 上 CoT 结构化 prompt 未带来增益
选型建议
  • 成本下界 / 快速基线 → Greedy（278 tokens, 2.1 s, 12.45%）
  • 最高准确率 → Best-of-N（23.35%，8× token）
  • 高 syntax + 结构化搜索 → ToT（98.4% syntax，16.8× token）
  • 失败自愈 / 按需额外调用 → Reflexion（+6 题挽救，10× token）
  • 采样 + 投票（需优化选优） → SC（oracle 24.9% ≫ pass@1 7.4%）
